# 重複推播規則說明

## 重複推播的工作原理

重複推播是一個有限次數的自動推播功能，**不會無止盡地重複**。

### 基本規則

1. **有限次數**: 推播會在達到指定的發送次數後自動停止
2. **時間間隔**: 每次推播之間有固定的時間間隔
3. **自動停用**: 完成所有推播後，系統會自動將推播狀態設為「停用」

### 參數限制

- **間隔時間**: 10秒 - 86400秒 (24小時)
- **發送次數**: 1 - 100次
- **總時長**: 不得超過7天
- **開始時間**: 必須是未來時間

### 推播生命週期

```
開始 → 第1次推播 → 等待間隔 → 第2次推播 → ... → 第N次推播 → 自動停用
```

### 狀態追蹤

系統會追蹤以下資訊：
- `sent_count`: 已發送次數
- `repeat_count`: 總計劃次數
- `is_active`: 推播是否仍在運行

當 `sent_count >= repeat_count` 時，推播會自動停用。

### 範例

如果設定：
- 開始時間：2025-01-15 09:00:00
- 間隔時間：60秒 (1分鐘)
- 發送次數：5次

推播時程：
- 第1次：2025-01-15 09:00:00
- 第2次：2025-01-15 09:01:00
- 第3次：2025-01-15 09:02:00
- 第4次：2025-01-15 09:03:00
- 第5次：2025-01-15 09:04:00
- 完成後自動停用

### 安全機制

1. **參數驗證**: 防止設定過短間隔或過多次數
2. **時長限制**: 總推播時長不超過7天
3. **自動清理**: 完成的推播會被標記為停用
4. **進度顯示**: 管理介面會顯示當前進度

### 與其他推播類型的區別

- **立即推播**: 播放1次後立即停用
- **定時推播**: 在指定時間播放1次後停用
- **重複推播**: 按間隔播放N次後停用
- **間隔推播**: 持續播放直到過期時間

### 故障排除

如果重複推播沒有按預期停止：
1. 檢查 `sent_count` 是否正確更新
2. 檢查 `updateNotificationSentCount` 函數邏輯
3. 檢查推播隊列過濾邏輯
4. 查看瀏覽器控制台和系統日誌

### 監控方式

- 管理介面顯示進度條和完成狀態
- 控制台輸出詳細的推播日誌
- 數據庫中的狀態欄位實時更新
