# 推播生命週期管理修復

## 問題描述
推播在正常播放完畢後沒有自動從播放清單中刪除，導致推播仍在執行的問題已修復。

## 修復內容

### 1. 推播自動停用邏輯
現在系統會根據推播類型自動處理停用：

- **立即推播**：播放完成後立即停用
- **定時推播**：到達指定時間播放完成後立即停用  
- **間隔推播**：持續運行直到過期時間到達才停用
- **重複推播**：發送次數達到設定值後自動停用

### 2. 推播隊列優化
- 增強推播隊列處理邏輯
- 添加詳細的日誌記錄
- 播放完成後自動更新推播狀態

### 3. 自動清理機制
- **定期清理**：每分鐘自動檢查並清理已完成的推播
- **手動清理**：推播管理頁面新增「清理已完成」按鈕
- **智能過濾**：獲取推播隊列時自動過濾無效推播

### 4. 過期推播處理
- 自動檢測過期推播並標記為停用
- 防止過期推播進入播放隊列

## 使用方法

### 測試推播生命週期
1. 執行 `test-broadcast-lifecycle.sql` 腳本
2. 觀察不同類型推播的行為
3. 驗證自動清理功能

### 手動清理已完成推播
在推播管理頁面點擊「清理已完成」按鈕，系統會：
1. 檢查所有啟用中的推播
2. 標記已過期或已完成的推播為停用
3. 重新載入推播列表

### 監控推播狀態
打開瀏覽器開發者工具的 Console，可以看到詳細的推播處理日誌：
- 推播開始播放
- 推播播放完成
- 狀態更新結果
- 自動清理活動

## 推播狀態說明

### 推播會被自動停用的情況：
1. **一次性推播**播放完成
2. **重複推播**達到設定次數
3. **任何推播**過期時間到達
4. **手動或自動清理**觸發

### 推播會繼續運行的情況：
1. **間隔推播**未到過期時間
2. **重複推播**未達到設定次數
3. **定時推播**尚未到達播放時間

## 技術細節

### updateNotificationSentCount 函數增強
```typescript
// 新的邏輯根據推播類型決定是否停用
if (notification.repeat_count && notification.repeat_interval_seconds) {
  // 重複推播：達到次數後停用
  shouldDeactivate = newSentCount >= notification.repeat_count
} else if (notification.interval_seconds) {
  // 間隔推播：過期才停用
  shouldDeactivate = notification.expires_at && now >= new Date(notification.expires_at)
} else {
  // 一次性推播：播放完成即停用
  shouldDeactivate = true
}
```

### 推播隊列過濾
```typescript
// 獲取推播隊列時自動過濾無效推播
const validNotifications = notifications.filter(notification => {
  // 檢查過期、完成狀態等
  return isValidNotification(notification)
})
```

## 預期行為

執行修復後，推播系統應該表現為：

1. ✅ 立即推播播放一次後自動消失
2. ✅ 定時推播到時間播放一次後自動消失
3. ✅ 重複推播按設定次數播放後自動消失
4. ✅ 間隔推播持續到過期時間才消失
5. ✅ 過期推播不會進入播放隊列
6. ✅ 推播管理列表及時反映實際狀態

## 故障排除

如果推播仍然沒有自動清理：

1. 檢查瀏覽器 Console 是否有錯誤訊息
2. 確認資料庫是否包含重複推播相關欄位
3. 檢查推播的 `is_active` 狀態是否正確更新
4. 手動執行「清理已完成」功能
5. 重新載入頁面清除快取

## 注意事項

- 修復後的系統向下相容，不影響現有推播
- 自動清理是安全的，只會停用推播而不刪除資料
- 推播狀態變更會在管理頁面即時反映
- 建議定期檢查推播管理頁面確認狀態正常
